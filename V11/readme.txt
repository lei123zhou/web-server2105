此版本开始完成对业务处理的支持
通过一个用户注册功能完成处理流程的实现。
用户注册的大致流程:
1:用户访问注册页面，并输入注册信息后点击注册按钮
2:浏览器将用户输入的数据提交给服务端
3:服务端解析提交的数据
4:根据解析出来的数据进行对应的注册操作
5:响应注册结果页面给浏览器

这里需要了解三方面的知识:
1:浏览器如何将用户在页面上输入的信息提交给服务端.表单的使用
2:服务端如何解析表单提交的数据。
  先学习GET请求提交表单数据的请求格式以及解析
3:如何处理业务的流程

实现:
1:在webapps/myweb下新建一个注册页面:reg.html
  这个页面上使用表单完成对用户注册信息的提交
  注意点:
  1.1:若希望将用户在页面上输入的信息提交给服务端，则必须使用form表单
  1.2:所有希望被提交的输入框都必须包含在form表单中
  1.3:每个输入框都必须使用name属性指定名字，否则提交时会被表单忽略
  1.4:表单提交到哪里是由form表单中action属性指定的值决定(注意页面中
      相对路径的位置)
  1.5:表单的提交需要submit提交按钮


2:重构HttpRequest解析请求行的操作
  由于表单使用get请求提交时，表单数据被包含在抽象路径中("?右侧内容")
  因此，我们需要对请求行中的抽象路径进行进一步解析拆分。并分别保存抽象
  路径中的请求部分和参数部分，以及每一组具体的参数以便后续处理请求时使用
  对此我们在HttpRequest中再定义三个属性:
  String requestURI     用于保存抽象路径中的请求部分(uri中"?"左侧)
  String queryString    用于保存抽象路径中的参数部分(uri中"?"右侧)
  Map parameters        用于保存每一组参数 key参数名 value参数值
                        注:参数名就是表单中输入框name属性的值
                           参数值为该输入框中用户输入的信息
